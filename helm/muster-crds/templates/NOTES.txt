The following CustomResourceDefinitions are now available in the cluster:

  - mcpservers.muster.giantswarm.io  (kind: MCPServer)

The CRDs carry `helm.sh/resource-policy: keep`, so they (and any
MCPServer / Workflow custom resources) survive `helm uninstall {{ .Release.Name }}`.

IMPORTANT: when using this chart, install or upgrade muster-crds BEFORE the
muster application chart so the CRDs are present before the operator starts.

The muster app chart also bundles these CRDs in its `crds/` directory, so they
are a no-op here if the app chart already created them. Use muster-crds when you
want CRD changes to apply on `helm upgrade` (Helm never updates a chart's
`crds/` directory on upgrade).

Verify the CRDs are registered:

  kubectl get crds mcpservers.muster.giantswarm.io workflows.muster.giantswarm.io
